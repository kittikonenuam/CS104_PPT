Design rationale

Short explanation for the presentation.

  • games stores the product shown to customers.
  • game_keys separates real stock keys from game details.
  • orders and order_items support one order with one or more games.
  • categories and platforms avoid repeated text and support filtering.
  • UNIQUE on email and game_key prevents duplicate users and duplicate activation keys.

SQL JOIN snippet

Dashboard query used to show game stock with category and platform.

SELECT
  g.game_id,
  g.gamename,
  g.price,
  COALESCE(c.cat_name, '-') AS category,
  COALESCE(p.platform_name, '-') AS platform,
  SUM(CASE WHEN k.sold = 0 THEN 1 ELSE 0 END) AS available_keys
FROM games AS g
LEFT JOIN game_categories AS gc ON g.game_id = gc.game_id
LEFT JOIN categories AS c ON gc.category_id = c.category_id
LEFT JOIN game_platforms AS gp ON g.game_id = gp.game_id
LEFT JOIN platforms AS p ON gp.platform_id = p.platform_id
LEFT JOIN game_keys AS k ON g.game_id = k.game_id
GROUP BY g.game_id;
